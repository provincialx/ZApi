## Что это такое?

Zed.dev — один из самых быстрых редакторов кода. У него встроенный AI-агент, который умеет читать файлы, запускать команды в терминале и писать код по вашим запросам. Но **сам Zed не содержит модели** — он просто стучится к OpenAI-compatible API.

ZApi — это мост между Zed и бесплатными веб-сервисами LLM (Qwen Chat, DeepSeek). Проект берёт на себя авторизацию, обход WAF и решение Proof-of-Work, а затем отдаёт ответы моделей по адресу `http://localhost:3264/api` в стандартном формате OpenAI.

**Результат:** Zed Agent получает доступ к мощным моделям — может читать вашу базу кода, править файлы и запускать терминал как полноценный агент. И всё это бесплатно через обычные веб-аккаунты.

**Это не скачанная модель на вашу видеокарту** и **не официальный API**. Это браузерный прокси с поддержкой нескольких провайдеров.

```text
Zed Agent → ZApi (localhost:3264) ─┬─ Qwen    (Puppeteer + обход WAF)
                                   └─ DeepSeek (PoW solver + HTTP fetch)
```

## Возможности агентов

| Провайдер | Статус агента | Детали |
|-----------|--------------|--------|
| **Qwen** | ✅ Полный агент | Tool calls, чтение/запись файлов, agent-loop, SSE. **Активно дорабатывается.** |
| **DeepSeek** | ⚠️ Только чтение | Может читать файлы через инструменты, но **не может их редактировать**. API позволяет, интеграция в процессе. |

## Как начать работу

Требуется установленный [Node.js](https://nodejs.org/) (версия 18+).

### 1. Установка и первый запуск
```bash
git clone https://github.com/provincialx/ZApi
cd ZApi
npm install
```

### 2. Запуск диспетчера
```bash
node index.js
```
Выберите провайдера:
- **1 — Qwen** (полноценный агент, обход WAF, мультиаккаунты)
- **2 — DeepSeek** (агент только для чтения, PoW, лёгкий)

### Настройка Qwen (рекомендуется для полного агента)

Добавьте аккаунт Qwen (откроется браузер Chromium — войдите на `chat.qwen.ai`, нажмите Enter):
```bash
node scripts/auth.js
# Меню → пункт 1 — Добавить новый аккаунт
```

Затем запустите с `SKIP_ACCOUNT_MENU=true` для фонового режима:
```bash
SKIP_ACCOUNT_MENU=true node index.js
# Выберите 1 — Qwen
```

### Настройка DeepSeek

Выберите пункт 2 — DeepSeek в меню диспетчера, затем:
1. **1 — Войти в DeepSeek** (откроется браузер — войдите через GitHub/Google)
2. Отправьте любое сообщение для загрузки PoW-модуля
3. Нажмите Enter. Сессия сохранится автоматически.
4. **3 — Запустить прокси**.

## Подключение к Zed.dev

1. Установите [Zed](https://zed.dev/).
2. Откройте настройки: `Cmd+Shift+,` → вкладка **AI**.
3. В поле **Base URL** введите: `http://localhost:3264/api`
4. В поле **API Key**: любой текст (например, `dummy-key`) — сервер не проверяет его
5. В поле **Model**: выберите модель (см. модели провайдеров ниже)
6. Сохраните. Теперь Zed Agent готов к работе.

### Рекомендуемые модели Qwen
- `qwen3-coder-plus` — лучший выбор для задач программирования
- `qwen3.7-max` — самая мощная, иногда игнорирует инструкции на "грязных" аккаунтах

### Модели DeepSeek
- `deepseek-v3` — быстрая (V4 Flash)
- `deepseek-r1` — режим рассуждения (thinking)
- `deepseek-expert` — экспертный режим (V4 Pro)
- `deepseek-v4-pro` — эксперт + рассуждения

> **⚠️ Важно для Qwen:** В профиле chat.qwen.ai → Персонализация — выключить все галочки, стиль "Кратко", очистить память.

## Работа с инструментами (Tool Calling)

Zed Agent может вызывать внешние инструменты по стандарту OpenAI tool calling: читать файлы, писать код, запускать команды в терминале. Модель сама решает какие инструменты нужны, вызывает их через Zed, получает результаты и продолжает работу — всё автоматически (agent loop).

### Qwen (кастомный prompt injection)
Веб-API Qwen Chat **не принимает инструменты** в стандартном формате OpenAI. ZApi обходит это:
1. Схемы инструментов впрыскиваются в сообщение пользователю как текстовые инструкции.
2. Qwen генерирует вызовы в виде JSON-блока внутри текстового ответа.
3. Прокси парсит блок, чистит лишний текст и отдаёт Zed чистые `tool_calls`.

Работает стабильно даже с большими списками инструментов (сжатие схем, лимит `MAX_SCHEMA_LEN=6000`, защита от зацикливания).

### DeepSeek (нативный OpenAI формат)
DeepSeek поддерживает стандартный параметр `tools[]` — прокси просто передаёт его насквозь. Никакого JSON-парсинга или инъекций не требуется.

## Мультиаккаунты и лимиты

Бесплатные аккаунты Qwen ограничивают количество запросов. ZApi поддерживает несколько аккаунтов:
- `node scripts/auth.js` → меню добавления/просмотра/удаления
- При лимите текущего аккаунта → сервер автоматически переключается на следующий (round-robin)
- Статусы: `OK` / `WAIT` / `INVALID`

DeepSeek использует одну сессию на инстанс.

## Полезные команды

| Команда | Описание |
|---------|----------|
| `node index.js` | Запустить диспетчер — выбор провайдера |
| `node scripts/auth.js` | Управление аккаунтами Qwen (добавить/список/удалить/перелогин) |
| `npm test` | Запуск 46 юнит-тестов (Node.js test runner, без браузера) |
| `npm run lint` / `format` | Проверка стиля кода (ESLint + Prettier) |

## Ограничения и нюансы

- **Неофициальный прокси.** Провайдеры могут менять структуру сайта или URL API в любой момент.
- **Qwen "грязная память".** Старые диалоги и персоназация могут заставить модель игнорировать инструкции. Очистите куки или переключитесь на `qwen3-coder-plus`.
- **Race condition ("in progress").** Qwen обрабатывает SSE-сессии несколько секунд после ответа. ZApi сам делает паузу (~1-2 сек) и повторяет запрос на тот же чат.
- **Память браузера (Qwen).** При долгих сессиях (>100 вызовов) Chromium потребляет больше RAM. GC закрывает страницы через 5 минут простоя, лимит 5 страниц, авторестарт при 512 MB RSS.
- **DeepSeek только чтение.** Может читать файлы, но пока не может их редактировать.
