# FreeQwenApi Proxy — Agent Brief (Session 2026-06-07)

## Архитектура
FreeQwenApi = браузерный прокси ForgetMeAI. Node.js + Puppeteer.
*   Заменяет веб-аккаунт Qwen Chat на локальный OpenAI-compatible API (`localhost:3264`)
*   Эмулирует вызовы Qwen через JS `fetch` внутри puppeteer-страницы
*   Zed/Aider -> FreeQwenApi -> Qwen Chat (через браузер) -> ответ обратно

## Найденные проблемы
### Сессия 1 (решены)
1. **`resolveQwenChatId` без return** → функция возвращала `undefined`, маппинг чатов не сохранялся, каждый запрос создавал новый Qwen-чат.
2. **Орфан-импорт посреди файла** (`import { testToken }`) → нарушало структуру JS модуля.
3. **«Потрошное месиво» в логах** (273 сообщения `user, assistant, tool...`) → бесконечная строка ролей забивала консоль и убивала контекст при agent-loop (tool_calls).
4. **Смерть сессий (`chat_not_exist`)** → Qwen удалял старые чаты, но прокси продолжал использовать их кэшированный `defaultForModel.chatId`. Результат: 500 Bad_Request.
5. **Неряшливый лог ошибок** (`undefined`) → `handleApiError` возвращал `{ error: undefined }`, что routes.js считал успехом (falsy check). Клиент получал пустой ответ вместо текста ошибки.

### Сессия 2 (решены сегодня)
6. **Отсутствие `protocolTimeout` в puppeteer.launch()** → дефолт 180с убивает большие задачи (редактирование файлов, длинные ответы Qwen). Ошибка: `Runtime.callFunctionOn timed out`.
7. **Бесконечный hang health-check пула страниц** → `page.evaluate()` без таймаута в `pagePool.getPage()`. Если CDP-соединение деградировало (страница жива, но JS-context повис), пул блокировался навсегда.
8. **Нет retry при сетевых ошибках goto** → `ERR_CONNECTION_RESET` на `CHAT_PAGE_URL` = мгновенный фейл без повторной попытки.
9. **Sliding window рвёт tool_call пары** → среза `.slice(-40)` мог оставить `role: "tool"` без предшествующего assistant с `tool_calls`, ломая контекст agent-loop.

| # | Фикс | Файл | Суть |

Добавлены хелперы: `invalidateModelDefaultChat()`, `isChatNotExistError()` |
| 4 | Sliding Window + компактные логи ролей | `routes.js` | Если история >60 сообщений, срезается до System Prompt + последние 40. Логирование бесконечной строки ролей заменено на сводку: `273 сообщений (1x system, 85x user...)`. Спасает от гигантских JSON-payload и падений Puppeteer. |

| **6** | **evaluate timeout 5s (Promise.race)** | **`chat.js`** | **Новый `evaluateWithTimeout()`: Promise.race(page.evaluate(), 5s). Если CDP-соединение зависло — страница быстро помечается мертвою, пул создаёт новую.** |
| **7** | **Retry goto с exponential backoff** | **`chat.js`** | **pagePool.getPage()重试 до 3 раз с задержкой RETRY_DELAY * attempt. Нестабильная сеть → больше не фейлит мгновенно.** |
| **8** | **Sliding window: сохранять tool_call пары** | **`routes.js`** | **После `.slice(-40)` проверяет: если первая оставшаяся message — `role: "tool"`, добавляет предшествующий assistant с tool_calls. Qwen видит полный контекст вызова.** |
